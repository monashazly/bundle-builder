# Prisma + SQLite Database

## When to use this skill

Read this before touching anything that involves:

- `prisma/schema.prisma`
- `prisma/seed.ts`
- `lib/prisma.ts`
- Any Prisma query inside a Route Handler
- Database migrations or setup commands

---

## Stack

- Database: SQLite (single file, zero setup, real SQL engine)
- ORM: Prisma (type-safe queries, migrations, seed command)
- Runtime: Next.js 15 App Router
- Language: TypeScript strict mode

## Why SQLite + Prisma for this project

SQLite requires no server, no account, no credentials, no Docker.
The entire database is one file: `prisma/dev.db`.
Prisma generates fully typed query results directly from the schema —
the same TypeScript types you use in components come from the same
source as what the database returns.

## Important: switching to MySQL or PostgreSQL in production

This project uses SQLite for zero-configuration local development.
Switching to a production database (MySQL, PostgreSQL) is a one-line
change in `schema.prisma` — the `provider` field and `DATABASE_URL` only.
All Prisma queries, all Route Handlers, all types remain identical.
This must be mentioned in the README.

```bash
npm install prisma @prisma/client
npm install -D tsx
npx prisma init --datasource-provider sqlite
```

This creates `prisma/schema.prisma` and adds `DATABASE_URL` to `.env`.

## Schema (prisma/schema.prisma)

This is the complete schema for this project. Do not modify it without
updating the seed script accordingly.

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

model Category {
  id        String    @id           // e.g. "cameras", "plan", "sensors", "extras"
  label     String                  // e.g. "Cameras", "Plan"
  stepIndex Int                     // 1, 2, 3, or 4
  products  Product[]
}

model Product {
  id             String    @id
  categoryId     String
  category       Category  @relation(fields: [categoryId], references: [id])
  name           String
  description    String
  image          String              // path e.g. "/images/cam-outdoor-pro.png"
  badge          String?             // nullable e.g. "Save 22%"
  compareAtPrice Float?              // nullable struck-through price
  price          Float
  learnMoreUrl   String?             // nullable
  position       Int                 // order within category (0-indexed)
  variants       Variant[]
}

model Variant {
  id        String  @id
  productId String
  product   Product @relation(fields: [productId], references: [id])
  label     String                  // e.g. "White", "Black"
  color     String                  // hex e.g. "#FFFFFF"
}

model Config {
  id        String   @id @default(cuid())
  createdAt DateTime @default(now())
  items     String   // JSON string — SQLite has no native JSON column type
                     // always use JSON.stringify() to write, JSON.parse() to read
}
```

## Prisma singleton (lib/prisma.ts)

Always import `prisma` from this file. Never write `new PrismaClient()`
anywhere else in the codebase.

Reason: Next.js hot reload in development creates a new module instance
on every file save. Without the singleton, each reload creates a new
PrismaClient, exhausting the SQLite connection limit quickly.

```ts
// lib/prisma.ts
// Prisma 6: generated client lives at lib/generated/prisma/client (not @prisma/client)
import { PrismaClient } from '@/lib/generated/prisma/client';

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

export const prisma =
  globalForPrisma.prisma ??
  new PrismaClient({
    log: process.env.NODE_ENV === 'development' ? ['error', 'warn'] : ['error'],
  });

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma;
}
```

## Seed script (prisma/seed.ts)

Inserts all product data. Must be idempotent — safe to run multiple times.
Always use `upsert`, never `create`, so re-running doesn't throw duplicate errors.

```ts
// prisma/seed.ts
// Prisma 6: import from generated path, not @prisma/client
import { PrismaClient } from '../lib/generated/prisma/client';
import { CATEGORIES } from '../lib/data/products';

const prisma = new PrismaClient();

async function main() {
  console.log('🌱 Seeding database...');

  for (const category of CATEGORIES) {
    await prisma.category.upsert({
      where: { id: category.id },
      update: {
        label: category.label,
        stepIndex: category.stepIndex,
      },
      create: {
        id: category.id,
        label: category.label,
        stepIndex: category.stepIndex,
      },
    });

    for (const [position, product] of category.products.entries()) {
      await prisma.product.upsert({
        where: { id: product.id },
        update: {
          name: product.name,
          description: product.description,
          image: product.image,
          badge: product.badge ?? null,
          compareAtPrice: product.compareAtPrice ?? null,
          price: product.price,
          learnMoreUrl: product.learnMoreUrl ?? null,
          position,
        },
        create: {
          id: product.id,
          categoryId: category.id,
          name: product.name,
          description: product.description,
          image: product.image,
          badge: product.badge ?? null,
          compareAtPrice: product.compareAtPrice ?? null,
          price: product.price,
          learnMoreUrl: product.learnMoreUrl ?? null,
          position,
        },
      });

      for (const variant of product.variants ?? []) {
        await prisma.variant.upsert({
          where: { id: variant.id },
          update: {
            label: variant.label,
            color: variant.color,
          },
          create: {
            id: variant.id,
            productId: product.id,
            label: variant.label,
            color: variant.color,
          },
        });
      }
    }
  }

  console.log(`✅ Seeded ${CATEGORIES.length} categories`);
  console.log(`✅ Seeded ${CATEGORIES.flatMap((c) => c.products).length} products`);
}

main()
  .catch((e) => {
    console.error('❌ Seed failed:', e);
    process.exit(1);
  })
  .finally(() => prisma.$disconnect());
```

Register the seed command in `package.json`:

```json
{
  "prisma": {
    "seed": "tsx prisma/seed.ts"
  }
}
```

## package.json scripts

```json
{
  "scripts": {
    "db:generate": "prisma generate",
    "db:migrate": "prisma migrate dev --name init",
    "db:seed": "prisma db seed",
    "db:setup": "npm run db:migrate && npm run db:seed",
    "db:reset": "prisma migrate reset --force",
    "db:studio": "prisma studio"
  }
}
```

| Command       | When to run                         | What it does                        |
| ------------- | ----------------------------------- | ----------------------------------- |
| `db:generate` | After every schema change           | Regenerates the TypeScript client   |
| `db:migrate`  | First time, or after schema changes | Creates/updates tables              |
| `db:seed`     | After migrate                       | Inserts product data                |
| `db:setup`    | First time setup                    | migrate + seed in one command       |
| `db:reset`    | When data is messy                  | Wipes everything and re-seeds       |
| `db:studio`   | Anytime                             | Visual DB browser at localhost:5555 |

## Prisma query patterns (used in Route Handlers)

```ts
// ─── Read ────────────────────────────────────────────────────────────────────

// Find all — ordered
prisma.category.findMany({
  orderBy: { stepIndex: 'asc' },
})

// Find all — with nested relations
prisma.category.findMany({
  orderBy: { stepIndex: 'asc' },
  include: {
    products: {
      orderBy: { position: 'asc' },
      include: { variants: true },
    },
  },
})

// Find one by id — returns null if not found
prisma.config.findUnique({
  where: { id: params.id },
})

// Find one — throws PrismaClientKnownRequestError if not found
// use inside try/catch
prisma.config.findUniqueOrThrow({
  where: { id: params.id },
})

// ─── Write ───────────────────────────────────────────────────────────────────

// Create — throws if id already exists
prisma.config.create({
  data: { items: JSON.stringify(items) },
})

// Upsert — use in seed scripts (safe to run multiple times)
prisma.product.upsert({
  where: { id: product.id },
  update: { name: product.name },
  create: { id: product.id, name: product.name, ... },
})

// Delete all — use in test cleanup
prisma.config.deleteMany()

// ─── JSON column pattern (SQLite has no native JSON type) ────────────────────

// Writing JSON
await prisma.config.create({
  data: { items: JSON.stringify(configItems) },
})

// Reading JSON — always cast the parsed result
const config = await prisma.config.findUnique({ where: { id } })
const items = JSON.parse(config.items) as ConfigItem[]
```

## Testing with Prisma

Use a dedicated test database so tests never corrupt development data.
Point to it via the `DATABASE_URL` environment variable in test setup.

```ts
// vitest.setup.ts
import { execSync } from 'child_process';

process.env.DATABASE_URL = 'file:./prisma/test.db';

// Run migrations once before all tests
// deploy is faster than dev — does not prompt for migration names
execSync('npx prisma migrate deploy', { stdio: 'pipe' });
```

```ts
// In each test file that touches the database
import { prisma } from '@/lib/prisma';
import { beforeEach, afterAll } from 'vitest';

// Clean mutable data before each test
// Never delete categories or products — they are read-only seed data
beforeEach(async () => {
  await prisma.config.deleteMany();
});

// Disconnect after the suite completes
afterAll(async () => {
  await prisma.$disconnect();
});
```

## Auto-setup check (optional — improves developer experience)

Add this to `lib/prisma.ts` to automatically run migrations if the
database file doesn't exist yet. This means `npm run dev` works on a
fresh clone without running `db:setup` first.

```ts
// lib/prisma.ts — extended with auto-setup
import { PrismaClient } from '@prisma/client';
import { execSync } from 'child_process';
import { existsSync } from 'fs';

const DB_PATH = './prisma/dev.db';

// Auto-migrate and seed if database file doesn't exist
if (!existsSync(DB_PATH)) {
  console.log('📦 Database not found — running setup...');
  try {
    execSync('npx prisma migrate deploy', { stdio: 'inherit' });
    execSync('npx prisma db seed', { stdio: 'inherit' });
    console.log('✅ Database ready');
  } catch (e) {
    console.error('❌ Auto-setup failed — run npm run db:setup manually');
  }
}

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

export const prisma =
  globalForPrisma.prisma ??
  new PrismaClient({
    log: process.env.NODE_ENV === 'development' ? ['error', 'warn'] : ['error'],
  });

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma;
}
```

With this in place, reviewer setup is just:

```bash
npm install && npm run dev
```
